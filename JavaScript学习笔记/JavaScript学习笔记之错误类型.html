JavaScript学习笔记 - 错误类型

发表于 2016-05-12 | 分类于 JavaScript学习笔记 |

Error 实例对象

JavaScript 在解析或者运行的时候发生错误，引擎就会跑出一个错误对象。JavaScript 原生提供Error构造函数，所有抛出的错误都是这个构造函数的实例。

JavaScript 语言标准中提到，Error实例必须有message属性，表示出错时的提示信息，没有提到其他属性。大多数 JavaScript 引擎，对Error实例还提供了name和stack属性。

  • message: 错误提示信息
  • name：错误名称（非标准属性）
  • stack: 错误的堆栈（非标准属性）

使用name和message这两个属性，能够对发生的错误有个大概的了解。

原生错误类型

Error实例对象是最一般的错误类型，Error还存在 6 个派生对象。

SyntaxError 对象

syntax,读音[ˈsɪntæks],意思是语句（语法），SyntaxError对象是解析代码时发生语法错误。看几个栗子：

ReferenceError 对象

reference，读音[ˈrefrəns]，意思是引用（参照），ReferenceError对象是引用一个不存在的变量时发生的错误。

另一个触发场景是，将一个值分配给无法分配的对象，比如函数的运行结果或者this赋值。

RangeError 对象

range，读音[reɪndʒ]，意思是范围。RangeError对象是一个值超出有效范围时发生的错误。常见的有数组长度为负数或者Number对象的方法参数超出范围，以及函数堆栈超过最大值。

TypeError 对象

TypeError对象是变量或参数不是预期类型时发生的错误。比如对字符串、布尔值、数值等原始类型使用new命令，就会抛出这种错误，因为new命令的参数应该是一个构造函数。

URIError 对象

URIError对象是 URI 相关函数参数不正确时抛出的错误，主要涉及encodeURI()、decodeURI()、encodeURIComponent()、decodeURIComponent()、escape()和unescape()这六个函数。

EvalError 对象

eval函数没有被正确执行，会抛出EvalError错误。该错误类型已不再使用，只是为了保证与以前代码兼容，才继续保留。

总结

以上 6 种派生错误，连同原始的Error对象，都是构造函数，开发者可以手动生成错误对象的实例。

try..catch 结构

一旦发生错误，程序就中止执行，JS 提供了try...catch结构，允许对错误进行处理，选择是否往下执行。
