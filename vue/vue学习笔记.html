Vue.js 简介

Vue.js 是什么

Vue.js 也称为 Vue,读音 /vju:/，发音类似 view
版本： v1.0 v2.0
官方网站

  • 是一个构建用户界面的框架
  • 轻量级的 MVVM（Model-View-ViewModel）框架，和 Angular、React 类似，其实就是所谓的数据双向绑定
  • 数据驱动 + 组件化的前端开发
  • 通过简单的 API 实现 数据响应式绑定 和 组合的试图组件
  • 更容易上手

vue 和 angular 的区别

angular

  • 上手较难
  • 指令以ng-xxx开头
  • 所有属性和方法存储在$scope中

vue

  • 简单、轻量级
  • 指令以v-xxx开头
  • HTML 代码+JSON 数据，创建一个 vue 实例

直接引用官方

CDN

直接下载到本地

可以直接下载vue.js到本地，引入到你的项目里边

NPM

在构建大型应用时推荐使用 NPM 安装，可以配合webpack和Browserify等模块打包使用

命令行工具(CLI)

官方提供了一个脚手架工具vue-cli，在进行单页面开发的时候可以进行快速搭建，新手学习不建议直接上手。

v-if/v-show

为什么把这两个指令放到一起呢？是因为这两个指令都能控制元素的显示/隐藏。不同的是v-show根据表达式的真假，切换元素的displayCSS 属性，来控制元素的显示和隐藏。v-if根据表达式的真假条件渲染元素，也就是如果表达式为false时，元素将不被渲染。看个例子：

结果如下：

通过这个例子我们可以看到，不管条件是什么(true or false)，v-show控制的元素总会被渲染，并且只是简单的基于display进行切换。而v-if则是真正的条件渲染，如果渲染时条件为假，那么它什么也不会做，直到条件变为真。

那么什么时候用v-if，什么时候用v-show。如果需要频繁的切换状态，那么使用v-show比较好，因为v-if有更高的切换开销，因为每次它都需要删除元素重新创建。如果运行条件很少改变时，那么使用v-if比较好，因为v-show拥有较高的初始渲染开销。
